I created this project -- originally called Projeto Peixes (Project Fishes) due to the initial intent, which later changed to Project Produtos (Products) - with some goals in mind:
The main focus is for this app to be continuously improving, be it through general reworks and quirks, or in its structure (by implementing new functions, updating the look & feel or through new techs).

This focus takes place in a web-app workspace, so I'm able to work with real-life utilities and progressively learn newer ways to achieve something.
Thus, I currently (as of April) use solely the base W3 Stack (HTML, JS, CSS) + PHP. No Laravel or React/Vue/Tailwind/etc. are intended to be used in this scope, as the goal is to master the fundamentals.
  Although the stack may be considered outdated (specially opting PHP over Node.js), it is useful to understand core concepts of server-side ops and how Js handles not only the front, but also
core operations at both sides even without Node itself; allowing me to get a even more solid grasp on concepts like async funcs, Promises and Callbacks and the http methods, understanding API core mechanisms.

Should I later focus on Next.js + Node.js, I'll get an advantage and actually enjoy all the improvements (as of routes and other specifics that make it easier) of those techs.
Speaking of which, I had started studying it before and I've got some projects on pause while I work my way through JS and PHP.

With Project Peixes, I was able to learn lots of new things, like the super-useful .map() function, DOM and JSON handling, fetch + http methods, template literals and string interpolation, events and async programming, regex,
AJAX, data input sanitizing and using page components (as the navigation bar and those .php files intended for server-side ops). I also went through a decent load of pure CSS to learn more and understand it before
putting everything on a frameworks' hand - I really dislike the verbose "class"-nature of these, at least for learning. I was able to use some CSS stuff on this project as I learned them:
CSS Variables, element's attributes inheritance (both with @extend [element] or the traditional listing).

I plan to start learning and implementing jQuery, after some more functionalities and tweaks take place.
